# 贡献代币（CT）系统规格说明

> 这是规格文档，不是实现。具体实现在 `implementations/` 目录。

---

## 核心设计原则

| 原则 | 说明 |
|------|------|
| 不可转让 | CT 只能通过贡献获得，不能买卖或赠送 |
| 历史衰减 | 历史 CT 每月减少 5%，防止早期贡献永久主导 |
| 非投机 | CT 不代表任何未来权益，只是治理权的量化工具 |
| 透明 | 每个成员的 CT 数量对所有成员实时可见 |

---

## 发行规则

**发行时机**：每个结算周期结束后（建议月度）

**发行量**：每期固定发行总量（例：每月 1000 CT）

**分配公式**：
```
成员X的本期CT = 1000 × (成员X的贡献分 / 所有成员贡献分总和)
```

---

## 贡献分计算维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 任务完成质量 | 40% | 人类评分 1-5 |
| 任务完成数量 | 30% | 已完成任务数 |
| 知识库贡献 | 15% | 有效知识条目数 |
| 成员引荐 | 10% | 被接受的引荐数 |
| 声誉维护 | 5% | 冲突处理、社群质量维护 |

---

## 衰减公式

```
当月有效CT = (历史CT × 0.95) + 本月新获CT
```

---

## 投票权计算

```
投票权比例 = min(个人CT / 总CT, 0.25)
```

单成员上限 25%，防止寡头控制。

---

## AI Agent 的 CT 处理

AI Agent 不直接持有 CT。其贡献计入"责任人"的贡献分。  
Agent 运行成本（API 费用）从运营成本中优先结算，不计入可分配利润。

---

## 最简实现路径

不强制区块链。最简单的实现：

1. **Notion 数据库**：手动记录贡献分，自动计算 CT（见 `implementations/notion-based/`）
2. **Google Sheets**：公式自动计算衰减和投票权
3. **GitHub Insights**：以贡献记录作为代理指标（适合技术项目）

链上实现见 `implementations/onchain-solana/`（待社区实现）
