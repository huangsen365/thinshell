## 所需 Notion 数据库结构

### 数据库 1：成员库（Members）

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名/代号 | 标题 | 成员标识 |
| 角色 | 选择 | 个人/团队/公司/Agent |
| 加入日期 | 日期 | |
| 当前CT | 数字 | 手动更新（每月结算后） |
| 投票权% | 公式 | `min(当前CT / [总CT], 0.25)` |
| 状态 | 选择 | 活跃/休眠/已退出 |

### 数据库 2：贡献记录（Contributions）

| 字段 | 类型 | 说明 |
|------|------|------|
| 成员 | 关联 | 关联到成员库 |
| 结算周期 | 日期 | 结算月份 |
| 任务质量分 | 数字 | 1-5，人类评分 |
| 任务数量 | 数字 | 本期完成任务数 |
| 知识贡献 | 数字 | 有效知识条目数 |
| 引荐数 | 数字 | |
| 贡献分合计 | 公式 | `质量×0.4 + 数量×0.3 + 知识×0.15 + 引荐×0.1` |
| 获得CT | 数字 | 结算后手动填入 |

---

## 月度结算流程（约 20 分钟）

```
1. 填写本月每个成员的贡献记录
2. 计算各成员贡献分占比
3. 按比例分配本期 1000 CT
4. 历史CT × 0.95（衰减）+ 本期获得CT = 新的当前CT
5. 更新成员库
6. 在 TSC 群里公示结果（截图即可）
```

---

## 注意事项

- 总CT 需手动维护（Notion 不支持跨数据库汇总公式）
- 建议每月固定一天结算，保持节奏
- 保留每期快照，便于事后审计
