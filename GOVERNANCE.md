## 权限层级

| 角色 | 条件 | 权限 |
|------|------|------|
| **任何人** | — | 提 Issue · RFC · PR · 参与讨论 |
| **贡献者** | 合并过 ≥1 PR 或采纳过 RFC | 标记 Issue，参与 RFC 投票（非约束性） |
| **核心维护者** | 6个月实质贡献 + 现有维护者邀请 | 合并 PR · 关闭 Issue · 主持 RFC |
| **创世维护者** | 项目发起人 | 维护创世层完整性（防止价值漂移） |

当前核心维护者：[@cat9999aaa](https://github.com/cat9999aaa)

---

## 决策流程

**日常 PR**（操作层修改）
```
提交 → 任意核心维护者 review → 无异议 48h → 合并
```

**RFC**（理论修改）
```
提交 → 公示 14 天 → 核心维护者投票（简单多数）→ 接受/拒绝
```

**白皮书重大修改**（宪法层等效）
```
提交 → 公示 30 天 → 核心维护者 2/3 多数 + 创世维护者确认 → 合并
```

## 版本号规则

```
v3.0.x   修复错误和矛盾
v3.x.0   合并重要 RFC（minor 升级）
v4.0.0   范式级理论突破（major 升级）
```

---

## 反腐化条款

触发维护者资格审查的行为：
- 打压反对意见
- 利用权限服务特定商业利益
- 未经讨论修改已通过的 RFC
- 删除历史记录（含被拒绝的 RFC 和关闭的 Issue）

---

## 年度宪法时刻

每年 1 月，核心维护者集体：
1. 确认过去一年演化方向符合创世层精神
2. 决定操作层规则续期 / 废除
3. 评估是否需要启动新 RFC

结果以 Discussion 形式公开记录。
