# 操作层规则模板

> 每条操作层规则都有有效期。超出未续期则**自动失效**（自动日落机制）。

---

## 规则卡片格式

```
规则编号：OP-[序号]
规则名称：[简短名称]
触发条件：[在什么情况下适用]
规定行为：[应该做什么 / 不能做什么]
日落日期：[YYYY-MM-DD，建议 3-12 个月]
提案者：[成员名称或代号]
通过日期：[YYYY-MM-DD]
```

---

## 示例规则

### OP-001：AI 输出外部交付前人工审查

| 字段 | 内容 |
|------|------|
| 触发条件 | 任何 AI Agent 生成的内容，准备发送给外部客户之前 |
| 规定行为 | 必须经过至少一名人类成员审查确认后方可发送 |
| 日落日期 | 2027-06-01 |

---

### OP-002：新成员接入 48 小时响应

| 字段 | 内容 |
|------|------|
| 触发条件 | 有新成员申请加入 TSC |
| 规定行为 | 核心委员会在 48 小时内给出明确的接受/拒绝回复 |
| 日落日期 | 2027-06-01 |

---

### OP-003：项目结束后 7 天内复盘

| 字段 | 内容 |
|------|------|
| 触发条件 | 每个项目完成交付后 |
| 规定行为 | 7 天内完成复盘文档（格式不限）并存入知识库 |
| 日落日期 | 2027-06-01 |

---

## 续期流程

接近日落日期 30 天前，系统自动提醒提案者。

续期提交 PR，内容未改则不需要重新投票，改动需重走投票流程。

未续期的规则在日落日期自动移入 `expired/` 目录，保留历史，不再有效。
