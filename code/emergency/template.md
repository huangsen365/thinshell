# 紧急协议模板

> 激活条件：核心委员会 2/3 同意，或 >50% CT 持有者请求  
> 最长有效期：**90 天自动终止**

---

## 适用触发条件

以下情况可激活紧急协议（勾选 TSC 适用项）：

- [ ] 关键任务严重偏差，客户关系面临重大风险
- [ ] 成员之间发生严重冲突，正常运作无法继续
- [ ] 关键技术基础设施失效
- [ ] TSC 公开输出包含严重错误，声誉受损
- [ ] [本 TSC 特有触发条件]

---

## 激活期间特殊规则

| 规则 | 说明 |
|------|------|
| 授权扩大 | 核心委员会可不经完整民主程序做决策（不含修改创世层） |
| 响应要求 | 收到信息请求后 24 小时内响应，延迟视为放弃意见权 |
| 审批加速 | 正常 7 天讨论期压缩为 24 小时 |

---

## 终止方式

| 方式 | 条件 |
|------|------|
| 自动终止 | 激活后 90 天 |
| 主动终止 | 核心委员会宣布危机结束 |
| 强制终止 | >60% CT 持有者要求 |

---

## 事后审查（终止后 30 天内必须完成）

1. 紧急协议期间的决策是否合理？
2. 需要修复哪些损害？
3. 是否需要修改操作层规则以防止同类危机？

结果以书面形式记录，存入 TSC 知识库。
